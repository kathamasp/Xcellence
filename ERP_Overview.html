คุณสมบัติเด่นของระบบ (Key Highlights)

Hybrid Partial Update (PATCH)
เปลี่ยนจาก PUT เป็น PATCH ทุกโมดูล ส่งเฉพาะฟิลด์ที่เปลี่ยน พร้อม Optimistic Locking ผ่าน updatedAt
Field-level Audit Log
บันทึก changeDetails ระดับฟิลด์ทุกครั้งที่มีการแก้ไข ผ่าน Audit Log Interceptor
3-Zone Dialog Layout
UI มาตรฐาน — Doc Info (Sticky Top), Form + Grid (Scroll Middle), Remark + Summary (Sticky Bottom)
Race Condition Free
GL pre-fetch นอก transaction + balance load ภายใน tx + batch findMany กัน N+1
Auto Stock Reservation
SO สร้าง → reserveStock อัตโนมัติ ตรวจ qtyOnHand - qtyReserved ก่อนยืนยัน
Property-Based Testing
7 correctness properties — DirtyTracker, Snapshot isolation, Optimistic Lock, JSON round-trip
📊 Outstanding Balance Pipeline 🆕
ระบบคงค้างทั้ง pipeline — SO→Issue→Ship Delivery→Ship Confirm พร้อม FIFO logic + real-time validation
👥 Contact Master Auto-Create 🆕
Contact = Master Entity หลัก — Customer/Vendor ถูกสร้างอัตโนมัติเมื่อตั้ง flag (cusFlag/supFlag) แบบ idempotent + transaction-safe
💰 Customer Price Resolution 🆕
บันทึกราคาลูกค้าจาก Sale Quotation Confirm — POST /resolve ค้นหาราคาที่มีผลบังคับใช้ ณ วันที่ + tie-breaker
🔒 Sale Order Lock by Quote Type 🆕
SO จาก SQ confirmed — ล็อคช่องราคา/จำนวนตาม specificQuotTypIdx (Type 1: Price · Type 2: Price+Qty)
💾 Global Save Behavior 🆕
Save ไม่ปิด dialog — อยู่หน้าเดิม reload ข้อมูลล่าสุด ทั้ง 18 หน้าเอกสาร
🚫 Soft-Delete Guards 🆕
ลบเอกสารได้เฉพาะ DRAFT/REVISED — Defense in Depth (Frontend hide + Backend reject 400)
🧾 Sale Invoice V2 Schema (TSaleInv) 🆕
ระบบ Sale Invoice ใหม่ใช้ Schema TSaleInv/TSaleInvProd/TSaleInvIsu — 8 endpoints + Auto-create AR + Save keeps screen open + Backward compat
🔍 saleFlag Filter (Sales Only) 🆕
โมดูลฝั่ง Sale แสดงเฉพาะสินค้าที่ saleFlag='Y' + stopSaleFlag='N' — กรองที่ database level (ไม่กระทบ Purchase/Inventory)
📊 AG Grid Row Grouping 🆕
Sale Order list — group ตามลูกค้า/สถานะ + Side Bar (Columns/Filters) + graceful degradation ถ้าไม่มี Enterprise license
🛒 Sale Order Direct Create + Source Pull 🆕
กดสร้าง SO เข้า Form ทันที (ไม่ผ่าน Wizard) + เลือก Sale Quotation/Customer P/O ใน Form → ดึง header + รายการสินค้า + ที่อยู่เปิดบิล มาใส่อัตโนมัติ
📥 Receive Module (ใบรับสินค้า) 🆕
โมดูลใหม่ TRcv/TRcvProd — Wizard 3 steps + Outstanding (key=lineOrder) + Exception Flag (สินค้านอกใบสั่ง) + Temp/Official docnum + Confirm validation
🧾 Supplier Invoice V2 (TSupInv) 🆕
Schema ใหม่ TSupInv/TSupInvProd/TSupInvPo — 8 endpoints + Tab Layout (PO refs + Products) + Pull Products จาก PO
💰 Set Sale Price (กำหนดราคาขาย) 🆕
หน้าใหม่ TChngSalePrice/Prod — 13 ประเภทเปลี่ยนราคา + Mark Down (ชั่วคราว/ถาวร) + Confirm → Upsert MProdPriceCom อัตโนมัติ
⚡ Auto PO Settlement 🆕
Confirm Receive ที่อ้างอิง SupInv + หมด outstanding → สร้าง PO Settlement อัตโนมัติ + Toast notification + Duplicate prevention + Atomic tx
🔍 purFlag Filter (Purchase Only) 🆕
โมดูลฝั่ง Purchase (SQ/PO/CPO V2) แสดงเฉพาะสินค้าที่ purFlag='Y' + stopPurFlag='N' + Default qty=1 + Auto-fill จากเอกสารต้นทาง
🏷 Price List (View-Only) 🆕
รวม Customer Price + Company Price เป็นหน้าเดียว 2 tabs (view-only) — การแก้ราคาทำผ่าน Set Sale Price แทน + GET /api/company-prices
📅 DateTime (Issue/Receive) 🆕
DB column isu_date/rcv_date จาก DATE → TIMESTAMP + Datepicker 24h format (บันทึกวันที่ + เวลาในใบเบิก/ใบรับ)
