  • Vendor.create() throw ForbiddenException
  • cusFlag='Y' → AutoCreateService.ensureCustomer (idempotent)
  • supFlag='Y' → AutoCreateService.ensureVendor (idempotent)
🔒 ห้ามลบเอกสารที่ไม่ใช่ DRAFT/REVISED
Defense in Depth — ทั้ง Frontend และ Backend ป้องกัน

  • BE: softDelete() guard statusTypIdx (5/6) → 400 Bad Request
  • ครอบคลุม Sale Order, Sale Quotation, Customer Requests
🚫 Outstanding Validation
ห้ามเบิก/ส่งเกินจำนวนค้างค้างที่เหลือ
  • Issue: ห้าม isuQty > outstanding ของ SO
  • Ship Delivery: ห้าม shipQty > outstanding ของ Issue (FIFO)
  • Ship Confirm: ห้าม confirm SD ที่ถูก confirm แล้ว (1:1)
  • Save form ก่อน confirm ทุกครั้ง
🔐 Sale Order Lock by Quote Type
SO ที่มาจาก SQ ที่ confirm — ล็อคช่องตามประเภท quote
  • specificQuotTypIdx = 1 (Price Quote): ล็อคราคา/ส่วนลด
  • specificQuotTypIdx = 2 (Price+Qty): ล็อคทุกอย่าง
  • ช่องล็อคแสดงพื้นหลังสีเทา
📅 Customer Price Expire Date
การจัดการช่วงวันที่ของราคาลูกค้า
  • null/blank → save เป็น 9999-12-31
  • Date range overlap query: Start_Date <= inputExpireDate AND Expire_Date >= inputStartDate
  • Tie-breaker: startDate DESC, lastChngDate DESC, xsysId DESC → top 1
